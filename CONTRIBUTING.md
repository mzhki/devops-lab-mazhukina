# Правила участия в проекте

Этот файл описывает соглашения и правила, которых необходимо придерживаться при работе с данным репозиторием.

---

## Структура репозитория

```
devops-lab-mazhukina/
├── README.md           # Описание репозитория
├── CONTRIBUTING.md     # Правила участия (этот файл)
├── LICENSE             # Лицензия
├── .gitignore          # Исключения для Git
├── images/             # Скриншоты и изображения для отчётов
├── lab0/               # Лабораторная работа №0
├── lab1/               # Лабораторная работа №1
├── lab2/               # Лабораторная работа №2
└── lab3/               # Лабораторная работа №3
```

---

## Ветки

| Ветка | Назначение |
|---|---|
| `main` | Стабильная версия. Содержит только проверенные и завершённые работы |
| `develop` | Рабочая ветка. Вся разработка ведётся здесь |

**Правило:** напрямую коммитить в `main` запрещено. Все изменения вносятся через `develop` и Pull Request.

---

## Именование веток

При необходимости создания дополнительных веток используется следующий формат:

```
lab<номер>/<краткое-описание>
```

Примеры:
- `lab1/docker-setup`
- `lab2/ci-pipeline`
- `lab3/grafana-config`

---

## Правила коммитов

Сообщение коммита должно быть на английском языке и чётко описывать, что было сделано.

**Формат:**
```
<тип>: <краткое описание>
```

**Типы:**
| Тип | Когда использовать |
|---|---|
| `feat` | Добавлена новая функция или файл |
| `fix` | Исправлена ошибка |
| `docs` | Изменения в документации или отчёте |
| `chore` | Вспомогательные изменения (настройки, зависимости) |

**Примеры:**
```
docs: add lab0 report
feat: add gitignore for python
chore: initial project setup
```

---

## Pull Request

1. Убедитесь, что все изменения закоммичены в ветке `develop`
2. Создайте Pull Request из `develop` в `main`
3. В описании Pull Request укажите:
   - Что было сделано
   - Какие файлы изменились
4. После проверки — смержите PR и удалите ветку `develop`

---

## Структура папки каждой лабораторной работы

Каждая папка `labN/` содержит:
- `README.md` — отчёт по лабораторной работе
- Дополнительные файлы, созданные в ходе работы (конфиги, скрипты и т.д.)

Скриншоты для всех лабораторных хранятся в общей папке `images/` в корне репозитория.  
Имена файлов должны быть без пробелов, формат: `lab<номер>-<описание>.png`

Пример: `lab0-github-profile.png`
